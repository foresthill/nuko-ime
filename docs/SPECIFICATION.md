# ぬこIME 開発仕様書

## 1. プロジェクト概要

### 1.1 背景
現存のIMEは海外または商業主義的なものが多く、日本人の開発効率や生産性を最大化する設計になっていない。日本人による日本人のためのIMEを開発することで、効率的な文字入力を実現し、ひいては日本人の開発効率向上、生産性向上、国力向上を目指す。

### 1.2 目的
- 日本語ユーザーの入力効率を向上させるIMEを開発する
- シンプルなロジックで使いやすいIMEを提供する
- フリー・オープンソースで提供し、多くの日本人ユーザーに使ってもらう

### 1.3 ターゲットユーザー
- 日本語を入力する全てのユーザー
- 特に開発者や文書作成を頻繁に行うユーザー
- オンラインコミュニティユーザー

## 2. 機能要件

### 2.1 基本機能
- 日本語かな入力からの漢字変換
- ローマ字入力対応
- 使用頻度に基づく変換候補の学習と表示順序の最適化
- Windows/Mac/Linux全OSでの動作

### 2.2 詳細機能
- **使用頻度学習**: ユーザーが選択した単語に重み付けを行い、次回からの変換候補の順位を上げる
- **文脈認識**: 前後の文脈から適切な変換候補を提示する（N-gramモデル + オプションでローカルLLM）
- **カスタム辞書**: ユーザー固有の用語や専門用語を登録できる機能
- **シンプルUI**: 最小限の操作で効率的に使えるインターフェース
- **予測変換**: 入力途中での変換候補提示
- **インライン変換**: 確定前の文字列をインラインで表示

### 2.3 差別化ポイント
- 純国産の日本人による日本人のためのIME
- Rustによるメモリ安全かつ高速な実装
- シンプルで軽量な設計（メモリ使用量最小化）
- ユーザーの入力パターンに素早く適応
- オープンソースコミュニティによる継続的改良
- プライバシー重視（完全オフライン動作可能）

## 3. 技術仕様

- **主要言語**: Rust（コア機能・OS統合・UI）
- **補助言語**: Python（ツール・スクリプト・辞書生成）

- **バージョン管理**: Git

### 3.6 データ構造

#### 学習データスキーマ
```rust
struct WordEntry {
    id: u64,
    surface: String,        // 表層形
    reading: String,        // 読み
    frequency: u32,         // 使用回数
    last_used: DateTime,    // 最終使用日時
    context_hash: u64,      // 文脈ハッシュ
}

struct ContextEntry {
    prev_word: String,      // 前の単語
    next_word: String,      // 次の単語
    count: u32,             // 出現回数
}
```


## 5. セキュリティ考慮事項

### 5.1 プライバシー保護
- ユーザーの入力データは端末内でのみ処理・保存
- ネットワーク通信は一切行わない（デフォルト）
- オプトインのみでの匿名統計データ収集

### 5.2 セキュリティ対策
- Rustの所有権システムによるメモリ安全性
- `#![forbid(unsafe_code)]` の積極的使用（OS統合部を除く）
- 依存クレートの定期的な監査（cargo-audit）
- コード署名によるバイナリ検証
- OSS開発によるコードの透明性確保

### 5.3 サプライチェーンセキュリティ
- 依存関係の最小化
- cargo-deny による依存関係ポリシー
- 再現可能ビルドの実現
